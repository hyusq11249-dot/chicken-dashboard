## 1. 한 줄 정의

> **"담당자가 매일 아침 한 화면에서 핵심 비즈니스 지표를 30초 안에 파악하고, 기간별 추이를 비교할 수 있게 한다."**

---

## 2. 타겟 & 문제

### 타겟 페르소나

| 페르소나 | 역할 | 핵심 페인포인트 |
|---------|------|----------------|
| **A — 팀장 / 매니저** | 주간 성과 보고 | 매주 여러 툴에서 수치를 취합해 리포트 작성에 1시간 소비 |
| **B — 마케터** | 캠페인 효과 측정 | 전환율·신규가입 상관관계를 한눈에 보기 어려움 |
| **C — 경영진** | 월간 현황 파악 | 실시간 매출·사용자 데이터를 즉시 확인할 수단이 없음 |

### 문제 정의

현재 팀은 **Google Sheets, 별도 분석 툴, Slack 수기 보고** 등을 조합해 KPI를 파악한다.
이 과정에서 ① 데이터 취합에 시간이 낭비되고 ② 기간 비교가 어려우며 ③ 보고 주기가 늦어진다.

## 3. 성공 지표

| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| **화면 로드** | 첫 KPI 카드 표시까지 **2초 이내** | Lighthouse / 직접 측정 |
| **일일 방문** | 출시 2주 후 팀원 80% 이상 자발 방문 | 접속 로그 |
| **보고 시간 절감** | 주간 리포트 작성 시간 50% 감소 | 팀 설문 |
| **데이터 정확도** | Mock → Supabase 전환 후 수동 검증 오차 0% | QA 체크리스트 |

## 4. 핵심 기능

### P0 — MVP 필수 (Day 1)

| 우선순위 | 기능 | 상세 | 완료 기준 |
|---------|------|------|-----------|
| P0-1 | **KPI 카드 4개** | 총매출 · 신규가입 · 활성사용자 · 전환율 | 카드 4개 렌더링, 전 기간 대비 증감률(%) 표시 |
| P0-2 | **라인 차트** | 기간별 매출 추이 | X축 날짜, Y축 금액, 툴팁 |
| P0-3 | **바 차트** | 기간별 신규가입 vs 활성사용자 비교 | 그룹 바, 범례 |
| P0-4 | **파이 차트** | 채널별 전환율 비중 | 레이블 + 퍼센트 |
| P0-5 | **기간 필터** | 주간 / 월간 / 분기 탭 전환 | 필터 변경 시 KPI·차트 동시 갱신 |
| P0-6 | **Mock 데이터** | 하드코딩된 JSON으로 전체 UI 구동 | Supabase 없이 완전한 화면 동작 |

### P1 — Day 2 (Supabase 전환)

| 기능 | 상세 |
|------|------|
| **Supabase 연동** | Mock JSON → Supabase 테이블 쿼리로 교체 |
| **실시간 갱신** | Supabase Realtime으로 KPI 자동 업데이트 |
| **날짜 범위 커스텀** | 시작일~종료일 직접 지정 |
| **CSV 내보내기** | 현재 뷰 데이터 다운로드 |
| **다크 모드** | 디자인 토큰 다크 레이어 적용 |

## 5. 비기능 요구사항

### 성능

| 항목 | 요구사항 |
|------|---------|
| 초기 로드 | FCP 2초 이하 (4G, Vercel 기준) |
| 필터 전환 | 기간 탭 클릭 후 **300ms 이내** 차트 갱신 (Mock 기준) |
| 번들 크기 | 초기 JS 200KB 이하 (gzip) |

### 반응형

| 브레이크포인트 | 레이아웃 |
|--------------|---------|
| Mobile (`< 768px`) | KPI 카드 2열, 차트 단일 컬럼 |
| Tablet (`768px – 1024px`) | KPI 카드 4열, 차트 2열 |
| Desktop (`1024px+`) | KPI 카드 4열, 차트 3열 |

### 보안

- Supabase Row Level Security (RLS) 활성화 (Day 2)
- 환경변수 `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` 분리 관리
- 민감 데이터(매출)는 인증된 사용자만 조회 (Day 2)

## 8. 데이터 모델

### KPI 카드

```typescript
interface KpiCard {
  id:       'revenue' | 'signups' | 'active_users' | 'conversion_rate';
  label:    string;
  value:    number;
  unit:     '원' | '명' | '%';
  change:   number;    // 전 기간 대비 증감률 (%)
  trend:    'up' | 'down' | 'flat';
}
```

### 차트 공통

```typescript
type Period = 'weekly' | 'monthly' | 'quarterly';

interface TimeSeriesPoint {
  date:   string;   // "2026-05-01" (ISO 날짜)
  value:  number;
}

interface GroupedPoint {
  date:     string;
  signups:  number;
  actives:  number;
}

interface PieSlice {
  channel: string;  // "Organic" | "Paid" | "Referral" | "Direct"
  value:   number;  // 퍼센트 (합산 100)
}
```

### Mock → Supabase 매핑 (Day 2)

| Mock 데이터 | Supabase 테이블 | 주요 컬럼 |
|-------------|----------------|-----------|
| 매출 시계열  | `daily_revenue` | `date`, `amount` |
| 가입/활성    | `user_events`   | `date`, `type`, `count` |
| 전환 채널    | `conversions`   | `channel`, `count`, `period` |
